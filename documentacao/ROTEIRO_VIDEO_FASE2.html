3. TECNOLOGIAS DO BACKEND 40 segundos
ANTÔNIO
"Para o backend, utilizamos as seguintes tecnologias:"
  • Node.js com Express - para criar a API REST
  • TypeScript - para tipagem estática e maior segurança
  • Prisma ORM - para comunicação com o banco de dados
  • PostgreSQL - como banco de dados relacional
  • JWT - para autenticação dos usuários
📺 Mostrar na tela: README.md ou slide com tecnologias
4. ARQUITETURA DO BACKEND 50 segundos
GABRIEL
"Agora vou mostrar como organizamos a estrutura do projeto."
"Seguimos uma arquitetura em camadas:"
  • Routes - define os endpoints da API
  • Controllers - recebe as requisições e valida os dados
  • Services - contém a lógica de negócio
  • Middlewares - autenticação JWT e validação
"A API possui 7 módulos: autenticação, usuários, questionário, emoções, metas, relatórios e recursos de autoajuda."
📺 Mostrar na tela:
1. VS Code com a pasta src/ expandida
2. Arquivo src/routes/index.ts mostrando os 7 módulos
5. BANCO DE DADOS 1 minuto
ANTÔNIO
"Agora vou mostrar a modelagem do banco de dados. Criamos 13 tabelas para atender todas as funcionalidades do app."
"As principais tabelas são:"
  • User - dados dos usuários com senha criptografada
  • Question e QuestionnaireResponse - questionário inicial com 8 perguntas
  • Emotion e EmotionRecord - catálogo de 20 emoções e registros
  • DailyCheckIn - check-in diário de humor, energia e sono
  • Goal e GoalProgress - metas de autocuidado
  • Report - relatórios semanais e mensais com insights
GABRIEL
"Também criamos tabelas de recursos de autoajuda com técnicas de respiração e meditação, e contatos de emergência como o CVV - 188."
